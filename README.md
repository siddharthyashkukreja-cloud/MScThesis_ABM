# MScThesis_ABM — Extended Chiarella–Bouchaud ABM in Python

This repository contains the code for my MSc thesis: a **Python-based agent‑based model (ABM)** of a financial market, built around an **extended Chiarella–Bouchaud (CB) trading model** with heterogeneous agents and a future **CCP risk layer**.[cite:19]

The project is structurally inspired by the **INET ESL** library (clean separation between core economic logic, simulation scaffolding, and Python bindings), but implemented directly in Python for transparency and modifiability in a thesis context.[cite:20][cite:21]

At the current stage, the focus is on the **trading core only**:

- A single risky asset
- Fundamental traders
- Momentum traders
- Noise traders
- A market/auctioneer implementing a linear price‑impact rule

Later stages will add:

- Clearing members (bank / non‑bank)
- CCP margining and default fund mechanics
- Default waterfalls and stress‑test scenarios (GFC vs non‑GFC)

---

## Project Goals

1. **Replicate and extend the Chiarella–Bouchaud class of models** in a clean Python ABM framework, with:
   - Nonlinear fundamental demand (cubic term in mispricing)
   - Momentum demand based on a smoothed price signal
   - Exogenous noise demand

2. **Calibrate the model** to SPY 5‑minute data (WRDS TAQ/IID) in both normal and stressed regimes, focusing on:
   - Price‑impact coefficient λ
   - Stylised facts (fat tails, volatility clustering, autocorrelation structure)

3. **Extend the model with a CCP layer**, inspired by my CCP Concentration Risk ODD document, to study how:
   - Margin calls
   - Default funds
   - Waterfall design and member heterogeneity  
   interact with the microstructure dynamics generated by the trading core.[cite:19]

## Trading Core: ABM Structure (Python Version)

### Time and State

- **Time step Δt:** One simulation step corresponds conceptually to a **5‑minute auction** in a continuous double auction (CDA), aligned with SPY 5‑minute intraday data.
- **Global state:**
  - Fundamental value \( v_t \)
  - Market price \( p_t \)
  - Momentum signal \( m_t \)
  - Random number generator state

These live in `model/globals.py` and `model/market.py`.

---

### `GlobalState` (planned `model/globals.py`)

Responsibilities:

- Store **calibration parameters**, such as:
  - \( \lambda \) — price‑impact coefficient
  - \( \kappa, \kappa_3 \) — fundamental demand coefficients
  - \( \beta, \gamma \) — momentum demand parameters
  - \( \sigma_n \) — noise trader volatility scale
  - Parameters for the fundamental process (\( \mu_v, \sigma_v \), Heston parameters in later stages)

- Implement the **fundamental value process**:
  - Initial value: \( v_0 \)
  - Update each step:
    - Start with a simple random walk / OU process in early stages
    - Later, move to:
      - Heston‑type stochastic volatility calibrated from SPY, or
      - A **data‑driven path** (e.g., embed a historical SPY/FTSE 5‑minute path)

Interface sketch (Python):

```python
class GlobalState:
    def __init__(self, params, rng):
        self.params = params
        self.rng = rng
        self.v = params.v0
        self.t = 0

    def step_fundamental(self):
        # Placeholder: simple diffusion / OU
        dv = self.params.mu_v * self.params.dt \
           + self.params.sigma_v * self.rng.normal()
        self.v += dv
        self.t += 1
```

### `Market` (planned `model/market.py`)

The `Market` class is the central auctioneer / price‑setting mechanism.

State:

- `price` — current market price \( p_t \)
- `fundamental` — current fundamental \( v_t \) (copied from `GlobalState`)
- `momentum` — EWMA of price changes \( m_t \)
- Statistics accumulators:
  - `excess_demand_t`
  - `cum_abs_demand`
  - `cum_excess_demand`

Core methods:

1. **Collect orders from traders**
   - Input: lists of `(side, volume)` from each agent at time \( t \).
   - Compute:
     - \( D_t = \sum_i q_i^+ \) (aggregate buy volume)
     - \( S_t = \sum_i q_i^- \) (aggregate sell volume)
     - \( ED_t = D_t - S_t \) (excess demand)

2. **Price impact update**
   - Linear price impact:
     \[
       \Delta p_t = \lambda \cdot ED_t
     \]
     \[
       p_{t+1} = \max(0, p_t + \Delta p_t)
     \]
   - This is the discretised form of the CB price‑impact relation with λ calibrated from TAQ/ESL‑style price impact estimates.

3. **Momentum update**
   - EWMA of price changes:
     \[
       m_{t+1} = \alpha (p_{t+1} - p_t) + (1 - \alpha) m_t
     \]

4. **Broadcast market information**
   - Returns a `MarketState` object with:
     - `price`
     - `fundamental`
     - `momentum`
   - Passed to all agents for their decision at the next step.

### Agents

All agents will derive from a simple Python `BaseTrader` class (planned `agents/base.py`).

#### `BaseTrader` (planned `agents/base.py`)

Common interface:

```python
class BaseTrader:
    def __init__(self, params, rng):
        self.params = params
        self.rng = rng

    def observe(self, market_state):
        """Update internal beliefs / signals."""
        raise NotImplementedError

    def decide(self, market_state):
        """
        Return (side, volume) where:
          side in {+1 (buy), -1 (sell), 0 (wait)}
          volume >= 0
        """
        raise NotImplementedError
```

This mirrors the role of `BaseTrader` in the Java prototype, but is now a simple Python protocol.

---

#### Fundamental Trader (`FundamentalTrader` — planned `agents/fundamental.py`)

Implements **mean‑reversion to fundamental value** with nonlinear demand.

Given market state \( (p_t, v_t) \):

- Mispricing:
  \[
    x_t = v_t - p_t
  \]
- Side:
  - If \( x_t > 0 \) → BUY (asset undervalued)
  - If \( x_t < 0 \) → SELL
  - Optionally 0 if within a small no‑trade band

- Demand magnitude:
  \[
    q^{(f)}_t = \kappa |x_t| + \kappa_3 |x_t|^3
  \]

This follows the extended Chiarella–Bouchaud specification with a cubic term capturing stronger reactions to large mispricings.

---

#### Momentum Trader (`MomentumTrader` — planned `agents/momentum.py`)

Implements **trend‑following** based on the momentum signal \( m_t \) broadcast by the market.

- Side:
  - If \( m_t > 0 \) → BUY
  - If \( m_t < 0 \) → SELL

- Demand magnitude (sigmoidal capping via `tanh`):
  \[
    q^{(m)}_t = \beta \tanh\big(\gamma |m_t|\big)
  \]
  where:
  - `β` sets the scale of demand,
  - `γ` tunes how quickly demand saturates.

---

#### Noise Trader (`NoiseTrader` — planned `agents/noise.py`)

Implements **zero‑intelligence / noise** order flow.

- Each step, a NoiseTrader:
  - Participates with probability \( p_{\text{order}} \).
  - If participating, chooses BUY or SELL with equal probability.
  - Uses a fixed volume \( \sigma_n \) (or small random variation around it).

This reproduces the background liquidity/noise component in CB‑style ABMs.

## Simulation Loop (Planned `model/simulation.py`)

High‑level pseudocode:

```python
for t in range(T):
    market_state = MarketState(price, fundamental, momentum)

    orders = []
    for trader in traders:
        trader.observe(market_state)
        side, volume = trader.decide(market_state)
        if volume > 0 and side != 0:
            orders.append((side, volume))

    price, momentum, stats = market.step(orders, global_state)

    global_state.step_fundamental()

    logger.record(t, price, fundamental=global_state.v, momentum=momentum, **stats)
```

This **trading‑only** loop will be the first thing implemented in Python. The CCP layer will later plug into the same loop by reading prices and positions and applying margin/default logic.

Planned extensions (later milestones):

   - Introduce a Heston or CIR‑like process for variance.
   - Calibrate to SPY 5‑minute WRDS IID data (normal vs GFC regimes).

   - Use different parameter files (`trading_gfc.yaml`, `trading_base.yaml`) to:
     - Increase λ, volatility and decrease liquidity in GFC regime.

   - `experiments/trading_core.py` for controlled runs of the trading model.
   - `experiments/calibration.py` for parameter estimation and stylised facts verification.

     - Price paths
     - Return distributions
     - Autocorrelations
